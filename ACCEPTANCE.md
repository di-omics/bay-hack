# Acceptance criteria: evidence before autonomy

The Track A claim is valid only if each biological, physical, and statistical
gate earns the next action.

## Evidence labels

- **modeled:** generated by a deterministic assay simulator
- **simulated execution:** real orchestration code against a simulator backend
- **measured:** returned by a physical reader, camera, or approved evidence file
- **hardware-validated:** measured data that also cleared the physical gates

No modeled value may be described as measured. No string supplied by a caller
may promote evidence to hardware-validated.

## TEM-1 gate table

| Stage | Acceptance criterion | Failure consequence | Current evidence |
|---|---|---|---|
| Protocol | Organizer-supplied expression method, substrate, readout, volumes, timing, compound sources, and confirmation are present | physical execution blocked | fail-closed configuration implemented |
| Protein synthesis | Replicated TEM-1 signal clears the organizer-defined fold threshold over no-template control and the configured CV limit | compound screen blocked | modeled path implemented, real evidence owed on-site |
| Round plan | Valid wells, required replicates, vehicle and no-enzyme controls, known compound IDs, positive dose factors, source wells, concentrations, units, and verified protocol | backend dispatch blocked | implemented and tested |
| Physical action | Plan passed, deck is clear, E-stop owner is named, human confirms motion, tip policy is safe | no motion | software gate implemented, venue confirmation owed |
| Kinetic evidence | Every assigned well has at least three unique time points and finite values | analysis blocked | CSV adapter implemented and tested |
| Assay quality | Vehicle-control slope exceeds no-enzyme background and Z-prime clears the declared minimum | data quarantined, no model update | implemented and tested |
| Candidate quality | Inhibition clears the declared threshold and replicate SD stays below its limit | candidate held | implemented and tested |
| Adaptive design | Round 1 QC passed and round 2 selection records the round 1 scores used | round 2 blocked | implemented and tested |
| Dose response | Round 2 QC passed, a condition is a hit, and the curve is monotonic within one-standard-error tolerance | nomination blocked | implemented and tested |
| Follow-up | A confirmed condition is nominated for downstream characterization | loop remains incomplete | implemented in simulation |
| Receipt integrity | Canonical payload matches its SHA-256 digest | replay blocked | implemented and tested |
| Stage replay | Receipt is valid and replay issues zero hardware commands | presentation blocked | implemented and tested |

## Expression refusal invariant

A failed expression gate must produce all five outcomes:

1. zero compound-screen execution
2. zero compound wells read
3. zero scientific-model updates
4. zero round 2 plans
5. zero robot commands after failure

The dashboard exposes this as **Prove expression refusal**.

## Round 1 to round 2 invariant

Round 2 must record:

- round 1 QC status
- the compound identifiers selected
- each mean inhibition value
- each standard error
- the conservative selection score
- the configured dose factors
- that observed round 1 evidence was used

A hand-edited second plate is not an adaptive loop.

## Dose-response claim boundary

The repository computes an uncertainty-aware monotonicity check and a relative
factor estimate for the 50 percent inhibition crossing. Until the organizer
supplies real concentrations and the data support a fitted pharmacology model,
do not call that value a definitive IC50.

## What is proven before the event

- Deterministic two-round TEM-1 simulation
- Balanced controls and replicated candidates
- Feature-diverse first-round selection
- Expression failure stopping every downstream action
- Kinetic slope extraction and source-file SHA-256
- Z-prime gate and failed-data quarantine
- Uncertainty-aware round 1 ranking
- Adaptive four-factor round 2 plan
- Dose-response monotonicity and relative 50 percent crossing
- Final nomination gate
- Tamper-evident receipt and zero-motion replay
- Generic liquid-handling fallback and its physical verification adapters
- Lazy seams to the di-omics robotics stack

## What is owed on-site

- Official protocol configuration reviewed by the track lead
- Real expression evidence
- Real compound library and source-well map
- At least one real robot-executed plate plan
- Real kinetic reader export
- Real round 1 control quality
- A round 2 plan generated from measured round 1 evidence
- Ideally, real round 2 confirmation
- Zeon simulation and physical execution trace
- Human motion confirmation and E-stop ownership
